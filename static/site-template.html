01
Signal / What changed?

Start with the work. Not the tool.

Requests, events, documents, and exceptions arrive from everywhere. A useful system first separates the signal that deserves action from the noise that does not.

  • Trigger
  • Intent
  • Priority
02
Context / What is true?

Bring the truth into the same frame.

Customer state, policy, history, ownership, and constraints have to travel with the request. Without context, even a fluent answer can create the wrong outcome.

  • State
  • History
  • Owner
03
Decision / What should happen?

Make judgment inspectable.

AI can interpret ambiguity, but evidence and operating rules define the choice. The decision must be explainable enough for an operator to challenge it.

  • Evidence
  • Rule
  • Judgment
04
Boundary / What may happen?

Capability is not authority.

Safe, reversible actions can pass. Uncertain, sensitive, or irreversible actions stop with a person who has the context and responsibility to decide.

  • Permission
  • Stop condition
  • Human gate
05
Action / What moves?

Move the work. Not just the text.

The approved decision updates the CRM, routes the case, schedules the work, or prepares the next handoff. Every action has an owner and a recovery path.

  • Tool
  • Handoff
  • Recovery
06
Evidence / What gets recorded?

Return proof to memory.

The system records what changed, why it changed, what was held, and who owned the exception. That evidence improves the next decision instead of disappearing into a chat log.

  • Record
  • Measure
  • Improve
